## Трамплин — платформа карьерных возможностей

Веб‑приложение на Django для поиска карьерных возможностей в IT: вакансии, стажировки, мероприятия и менторские программы.  
Сервис поддерживает три основных роли пользователей: **соискатель**, **работодатель** и **куратор**.

- **Соискатели**: создают публичный профиль, откликаются на вакансии и стажировки, сохраняют избранные предложения, получают рекомендации и могут просить о менторстве.
- **Работодатели**: оформляют страницу компании, публикуют и редактируют вакансии/стажировки/мероприятия, обрабатывают отклики, работают с отзывами о компании.
- **Кураторы/админы**: модерируют вакансии, отзывы и заявки на менторство, верифицируют работодателей, управляют блокировками пользователей.

Бэкенд написан на **Django**, в качестве базы данных используется **MySQL**. Для быстрого развёртывания есть готовая конфигурация **Docker / docker‑compose** и SQL‑дамп `tramplin.sql` с тестовыми данными.

---

## Стек технологий

- **Язык**: Python 3.12
- **Фреймворк**: Django
- **База данных**: MySQL 8.x (или совместимая MariaDB)
- **Веб‑сервер (dev)**: встроенный `runserver`
- **Контейнеризация**: Docker, docker‑compose

---

## Структура проекта (кратко)

- `config/` — настройки Django‑проекта, `settings.py`, `urls.py`, WSGI.
- `tramplin/` — основное приложение:
  - модели пользователей, вакансий, откликов, чатов, отзывов и т.д.;
  - формы регистрации/логина и редактирования профилей;
  - вьюхи для личных кабинетов соискателя, работодателя и куратора;
  - публичные страницы компаний и профилей.
- `tramplin.sql` — дамп БД MySQL с актуальной схемой и тестовыми данными.
- `Dockerfile`, `docker-compose.yml` — окружение для запуска через Docker.

## Быстрый старт через Docker

Требования:

- установлен **Docker** и **docker‑compose**.

1. Скопируйте `.env.example` (если есть) в `.env` и при необходимости задайте значения:
   - `SECRET_KEY` — секретный ключ Django;
   - `DEBUG` — `True` / `False`;
   - `DB_NAME`, `DB_USER`, `DB_PASSWORD` и др.
2. В корне проекта (`else/`) выполните:

```bash
docker-compose up --build
```

3. После успешного старта:
   - БД MySQL поднимется в контейнере `db` и заполнится дампом `tramplin.sql`.
   - Django‑приложение будет доступно по адресу `http://localhost:8000/`.

Остановка контейнеров:

```bash
docker-compose down
```

## Локальный запуск без Docker

Требования:

- Python 3.12;
- установленный сервер MySQL / MariaDB.

1. Клонируйте репозиторий и перейдите в папку проекта:

```bash
git clone <URL_репозитория>
cd if_else/else
```

2. Создайте и активируйте виртуальное окружение:

```bash
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # Linux/macOS
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Создайте базу данных `tramplin` в MySQL и импортируйте дамп:

```bash
mysql -u root -p tramplin < tramplin.sql
```

5. Установите переменные окружения (либо через `.env`, либо вручную), чтобы они совпадали с настройками из `config/settings.py`:

- `DB_NAME=tramplin`
- `DB_USER=<пользователь MySQL>`
- `DB_PASSWORD=<пароль>`
- `DB_HOST=localhost`
- `DB_PORT=3306`

6. Выполните миграции (на случай, если структура изменилась после дампа):

```bash
python manage.py migrate
```

7. Запустите дев‑сервер:

```bash
python manage.py runserver
```

Приложение будет доступно по адресу `http://127.0.0.1:8000/`.

## Роли и основные возможности

- **Соискатель**:
  - регистрация и авторизация;
  - заполнение профиля (навыки, университет, портфолио и т.д.);
  - отклики на вакансии и стажировки;
  - сохранение избранных возможностей;
  - запросы на менторство и получение рекомендаций.

- **Работодатель**:
  - создание и редактирование страницы компании;
  - публикация вакансий, стажировок и мероприятий;
  - просмотр и фильтрация откликов;
  - управление статусами откликов;
  - получение и модерация отзывов о компании.

- **Куратор / админ**:
  - панель модерации пользователей;
  - верификация работодателей;
  - модерация отзывов и объявлений;
  - работа с заявками на менторство;
  - блокировка / разблокировка аккаунтов с логированием действий.

python manage.py migrate

## Замечания по безопасности

- Файл `.env` содержит чувствительные данные и **не должен** попадать в репозиторий.
- В продакшене обязательно:
  - установить уникальный `SECRET_KEY`;
  - выключить `DEBUG` (`DEBUG=False`);
  - настроить `ALLOWED_HOSTS` под ваши домены;
  - использовать отдельный, защищённый экземпляр MySQL.
